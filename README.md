### How This Works

Each investigation you complete in the SOC Simulation lab gets its own report file — either the Markdown template (good for GitHub rendering) or the Word doc (good for filling in, converting to PDF, and attaching). Drop them into the relevant folder, update the index table below, and your portfolio builds itself over time.

```
soc-investigations/
├── README.md
├── alerts/
│   ├── TEMPLATE.md
│   └── [your completed alert investigations]
└── bec/
    ├── TEMPLATE.md
    └── [your completed BEC investigations]
```

## What This Portfolio Demonstrates

This repository contains documented investigations completed in the **SOC Simulation Project** — a live, lab-driven training environment built on Microsoft Defender for Endpoint and Microsoft Sentinel, with real alerts generated by emulated threat actor TTPs.

The investigations here demonstrate practical ability in:

- **Threat investigation** across endpoint and identity using enterprise security tooling
- **DFIR methodology** — structured investigative thinking applied to real alert and incident scenarios
- **Microsoft Sentinel & Defender for Endpoint** — KQL-based log analysis, alert triage, timeline reconstruction
- **BEC investigation** — Unified Audit Log analysis, IP triage, session ID pivoting, data access identification
- **Documentation** — translating raw investigation notes into structured, readable reports

> The focus throughout is investigative methodology over tool memorisation — the same approach used in real incident response engagements.
